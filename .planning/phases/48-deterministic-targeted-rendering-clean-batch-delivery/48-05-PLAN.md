<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| canonical Markdown → publication facts | canonical 可含 declined/pending 与候选 ID；只有 verified 且 grammar 合法的事实可越界。 |
| structured facts → layout/Typst | 映射、列表与字符串是 candidate-controlled 内容，必须按 allowlist 序列化并统一测量/发射。 |
| final Markdown/layout → Typst compiler | 即使前置校验失效，container ID 与文本也不能闭合 Typst 字符串或执行表达式。 |
| photo mode/assets → candidate/current | 显式模式与本地路径均不可信；缺失、symlink、unknown 或类型错误必须在发布前失败。 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-48-G05-01 | Elevation of Privilege | fact ID → Typst | mitigate | 单一稳定 grammar 在 publication/final load/emitter 三层验证，emitter 仍调用 `typst_content` 二次转义；恶意 ID 端到端负例。 |
| T-48-G05-02 | Information Disclosure | declined facts → deliverables | mitigate | canonical 与 publication view 分离，只有 verified facts 可进入 projection；覆盖 declined overrides 和三种产物文本断言。 |
| T-48-G05-03 | Tampering | structured display serializer | mitigate | 每模块字段 allowlist、只接受字符串/字符串列表/受控映射、控制字符和未知嵌套 fail closed。 |
| T-48-G05-04 | Spoofing | explicit photo mode | mitigate | photo/no-photo/auto 先分支，显式 photo 必须绑定 provided 安全资产；无资产公开负例。 |
| T-48-G05-05 | Tampering | assets/delivery paths | mitigate | 复用 no-follow、regular-file、root containment 与 candidate-first；symlink/unknown 失败 snapshot 不变。 |
| T-48-G05-SC | Tampering | package supply chain | accept | 不新增 npm/pip/cargo 安装；仅复用仓库已有 Typst、PyMuPDF 与 Pillow。 |
</threat_model>
