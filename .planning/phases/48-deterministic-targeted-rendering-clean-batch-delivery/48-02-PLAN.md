<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| final Markdown text → YAML/Typst | 候选人和招聘文本可包含 YAML/Typst 元字符，必须保持数据语义。 |
| local photo bytes → embedded Typst | 本地照片可能是 symlink、伪格式、含 EXIF/路径或在读取间被替换。 |
| dynamic identity fields → filenames | 姓名、单位、岗位、主题经规范化后可能泄密、穿越目录或碰撞。 |
| candidate matrix → delivery service | partial triple 或失败主题不得被误标为可发布集合。 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-48-05 | Tampering | Markdown/YAML/Typst emitters | mitigate | safe_load 精确 schema、round-trip hash/ID 校验、所有 Typst 内容统一转义，注入 fixture 回归。 |
| T-48-06 | Information Disclosure | photo normalization/emission | mitigate | no-follow bounded read、格式验证、去路径/EXIF、内嵌 bytes；扫描正式 `.typ`/YAML 不含源路径和 EXIF。 |
| T-48-07 | Tampering | safe stem builder | mitigate | NFKC、字符/长度 allowlist、禁止路径分隔符和敏感字段、全矩阵碰撞预检后才写 candidate。 |
| T-48-08 | Denial of Service | render matrix | mitigate | 字面三主题 registry、受控页数/输入大小、subprocess 返回码与非空输出检查，失败不产生可发布矩阵。 |
| T-48-09 | Information Disclosure | evidence/public split | mitigate | trace、condition JSON、preview、normalized photo 只在 owned work evidence；candidate exact-set 测试拒绝额外文件。 |
| T-48-SC | Tampering | package supply chain | accept | 不新增依赖；使用仓库已验证 PyYAML、Typst 0.15 与 stdlib，计划不执行 package install。 |
</threat_model>
