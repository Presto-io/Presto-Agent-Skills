<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| delivery root entries → managed discovery | 根目录可能包含未知、symlink、partial、stale 或伪装普通文件。 |
| candidate triples → mutation set | 动态 filenames/path set/bytes 决定 replace、archive 与 delete。 |
| preflight snapshot → publication | 用户确认后根目录可能被交换或内容发生 TOCTOU 变化。 |
| signal/error → rollback | 中断可能发生在 history、replace、unlink 或 post-verify 任一阶段。 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-48-10 | Tampering | root/current discovery | mitigate | O_NOFOLLOW、held dev/inode、descriptor-relative stat/read、exact triple grouping；unknown/symlink/partial/stale fail closed。 |
| T-48-11 | Elevation of Privilege | replace/unlink targets | mitigate | 只允许显式 safe managed names，在 held root dir_fd 下操作，支持目录也禁止 symlink。 |
| T-48-12 | Repudiation | removed/updated history | mitigate | approved delta 绑定 unchanged no-follow snapshot；history 按 stem 保存旧完整 triple并 post-validate。 |
| T-48-13 | Denial of Service | error/INT/TERM mid-publication | mitigate | 预写 rollback、信号 handler、fault-point tests、全 current snapshot 恢复与 exact post-verify。 |
| T-48-14 | Information Disclosure | root pollution | mitigate | evidence/preview/normalized photo 仅 `.work`；root allowlist 和 exact managed set 测试阻止泄露。 |
| T-48-SC | Tampering | package supply chain | accept | 纯 stdlib 文件系统实现且不新增包安装。 |
</threat_model>
