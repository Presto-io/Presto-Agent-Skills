<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Markdown comment → verified facts | 候选人可控 metadata 可能缺失、拼错、重复或覆盖 review 状态。 |
| canonical path → publication snapshot | 本地路径可为 symlink/特殊文件，或在加载后被替换。 |
| parsed facts → approval digest | 若事实与 hash 来自不同读取，审批会绑定错误输入。 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-48-G07-01 | Spoofing | `parse_metadata` | mitigate | exact key set、重复键/重复 comment 拒绝、无 verified 缺省。 |
| T-48-G07-02 | Tampering | canonical input | mitigate | `O_NOFOLLOW` + regular-file `fstat` + bounded single read + frozen SHA-256。 |
| T-48-G07-03 | Repudiation | final metadata / approval | mitigate | final Markdown 与 approval 共用 `ResumeDocument.source_sha256`。 |
| T-48-G07-04 | Denial of Service | canonical reader | mitigate | 固定最大字节数、无效 UTF-8 与特殊文件在 mutation 前拒绝。 |
| T-48-G07-SC | Tampering | package supply chain | accept | 不新增包安装、联网或外部依赖。 |
</threat_model>
