<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| existing delivery root → session | 根目录可能含私密 sources/assets、unknown、symlink 或 stale work。 |
| transaction cleanup → public result | 文件系统可拒绝 unlink/rmdir/close，导致残留资源与错误成功状态。 |
| publication failure → cleanup failure | 双重失败必须保留 rollback 状态和异常因果，不能互相隐藏。 |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-48-G09-01 | Information Disclosure | delivery root | mitigate | sources/assets 一律 unknown，诊断与私密附件不能被合法 current 吸收。 |
| T-48-G09-02 | Elevation of Privilege | support directories | mitigate | 只允许 held/no-follow history 与 exact owned `.work`。 |
| T-48-G09-03 | Repudiation | cleanup result | mitigate | 聚合稳定阶段/errno 并返回非零，不再吞掉 OSError。 |
| T-48-G09-04 | Denial of Service | stale `.work/.lock` | mitigate | cleanup 失败立即可见；残留保持 fail-closed 供审计。 |
| T-48-G09-05 | Tampering | double failure | mitigate | rollback 先恢复 current，cleanup error 通过异常链保留原始 publication failure。 |
| T-48-G09-SC | Tampering | package supply chain | accept | 不新增包安装或网络依赖。 |
