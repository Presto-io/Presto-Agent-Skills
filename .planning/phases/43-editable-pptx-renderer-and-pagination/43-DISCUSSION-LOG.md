## 正文与代码如何拆页

### 多块装填

| Option | Description | Selected |
|--------|-------------|----------|
| 顺序装填，必要时才拆块 | 按源码顺序装满当前页；仅超大单块内部拆分 | ✓ |
| 每个顶层块独占一页 | 规则简单但产生较多空白页 | |
| 跨页均衡分配 | 视觉均衡但分页不够直观 | |

**User's choice:** 顺序装填，必要时才拆块。

### 超大段落和列表

| Option | Description | Selected |
|--------|-------------|----------|
| 优先语义边界，极端情况硬拆 | 段落按句子、列表按列表项，极端情况才硬拆 | ✓ |
| 只按段落和列表项拆分 | 超大句子或列表项直接失败 | |
| 统一按字符预算拆分 | 稳定但可能产生生硬断点 | |

**User's choice:** 优先语义边界，尽量不要在句子中间拆开。

### 长代码块

| Option | Description | Selected |
|--------|-------------|----------|
| 完整代码行分页，超长行软换行 | 不修改代码文本，按显示行计预算 | ✓ |
| 完整代码行分页，超长行缩小字号 | 到最小字号仍放不下则失败 | |
| 允许拆分超长代码行 | 容纳能力强但损害可读和复制 | |

**User's choice:** 完整代码行分页，超长行软换行。

### 小标题续页

| Option | Description | Selected |
|--------|-------------|----------|
| 每个续页片段重复小标题 | 保留上下文，不添加“续” | ✓ |
| 只在第一个片段显示 | 忠实出现次数但续页缺少语境 | |
| 续页显示弱化小标题 | 需要额外视觉状态 | |

**User's choice:** 每个续页片段重复原小标题。

## 表格、时间线与目录续页

### 表格续标记

| Option | Description | Selected |
|--------|-------------|----------|
| 放在表格标题层 | 有表名时追加“（续）”，主标题不变 | ✓ |
| 加到幻灯片主标题 | 与 Phase 42 标题规则冲突 | |
| 只重复表头 | 不满足续标记验收 | |

**User's choice:** 表名和表格本体组成一个复合块；表名可缺省，并且与幻灯片二级标题无关。有表名才追加“（续）”。

### 表格视觉孤行

| Option | Description | Selected |
|--------|-------------|----------|
| 整行不可跨页 | 单元格换行并保持数据行完整 | ✓ |
| 允许一行跨页 | 损害原生表格编辑体验 | |
| 持续缩小直到放下 | 可能越过可读字号边界 | |

**User's choice:** 控制物理显示上的孤行，不是逻辑数据孤行；不能报错或截断。必要时可调大字号促使孤行重新换行为多行。

### 时间线孤项

| Option | Description | Selected |
|--------|-------------|----------|
| 弹性全局均衡 | 保持顺序并重排全部相关页 | ✓ |
| 严格按容量装填 | 最后一页可只剩一个节点 | |
| 固定每页节点数 | 无法适应长说明 | |

**User's choice:** 每页弹性容纳若干节点；下一页只剩一个时要么吸回前页，要么再移两个过去。上上一页等更早页面也要同步调整，使所有页面视觉重量接近；正常每页至少三个节点。

### 目录分页

| Option | Description | Selected |
|--------|-------------|----------|
| 弹性分页并控制孤项 | 连续编号、全局重排、无续标记 | ✓ |
| 先缩小字号保持单页 | 可能使目录整体过小 | |
| 固定每页条目数 | 不适应长标题换行 | |

**User's choice:** 弹性分页。各页条目数量必须尽量接近，再按实际显示高度微调，不能出现多的多、少的少。

## 可编辑对象的最低标准

### 表格对象

| Option | Description | Selected |
|--------|-------------|----------|
| 原生 PPTX 表格 | 数据保持原生行列编辑能力 | ✓ |
| 文本框和矩形拼表格 | 视觉可控但失去表格语义 | |
| 按复杂度混用 | 编辑体验不一致 | |

**User's choice:** 原生 PPTX 表格。

### 时间线结构

| Option | Description | Selected |
|--------|-------------|----------|
| 所有对象完全独立 | 兼容稳定但整体移动繁琐 | |
| 每个节点为组合对象 | 节点整体移动，内部仍可编辑 | ✓ |
| SmartArt | 工具链和跨查看器稳定性不足 | |

**User's choice:** 每个节点组合更好；主轴独立，节点组合内部不得栅格化。

### 图集结构与空槽

| Option | Description | Selected |
|--------|-------------|----------|
| 图片与图注组成组合对象 | 保持相对位置，内部仍可编辑 | ✓ |
| 图片和图注独立 | 移动时容易错位 | |
| 图注写入替代文本 | 不再是可见图注 | |

**User's choice:** 图片和图注组成组合对象；没有图注也保留空图注框。表格没有表名时同样保留空白占位符。

### 粗体与高亮

| Option | Description | Selected |
|--------|-------------|----------|
| 原生文本 run | 连续文本仍可编辑、复制 | ✓ |
| 只保留纯文本 | 丢失已定义语义 | |
| 独立文本框和底色形状 | 容易错位 | |

**User's choice:** 使用原生文本 run。

## 渲染失败与输出覆盖策略

### 无效输入产物

| Option | Description | Selected |
|--------|-------------|----------|
| 默认不生成，显式请求诊断 | 正式与诊断边界最清楚 | |
| 始终生成隐藏诊断文件 | 排错方便但不直观 | |
| 公开生成 best-effort PPTX | 用户可立即查看和手工修正 | ✓ |

**User's choice:** 公开生成 best-effort PPTX，并要求 agent 明确提示异常；命令仍非零退出。

### 异常文件标识

| Option | Description | Selected |
|--------|-------------|----------|
| 使用请求的正式文件名 | 不加水印，靠 CLI 和 agent 明示异常 | ✓ |
| 自动改成诊断文件名 | 不易误用但改变目标路径 | |
| 加入可见警告页 | 明确但污染内容 | |

**User's choice:** 使用请求的正式文件名。用户修正后覆盖异常 PPTX，或在自动工作流结束后手工修改。

### 覆盖策略

| Option | Description | Selected |
|--------|-------------|----------|
| 原子覆盖 | 临时生成成功后一次替换，失败保留旧文件 | ✓ |
| 必须 `--force` | 修订循环繁琐 | |
| 自动备份 | 污染公开目录 | |

**User's choice:** 原子覆盖。

### 标准公开产物

| Option | Description | Selected |
|--------|-------------|----------|
| 仅显式请求时复制 Markdown | 默认只有 PPTX | |
| 每次同时输出 Markdown | 固定双产物交付 | ✓ |
| 永不复制 Markdown | 不保留可编辑源 | |

**User's choice:** 标准产物就是 Markdown 和 PPTX 两个文件。
