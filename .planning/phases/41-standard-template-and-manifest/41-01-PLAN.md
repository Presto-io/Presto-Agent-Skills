<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| POTX evidence -> normalized PPTX | Binary Office package from `test/学院PPT模板.potx` is used as source evidence and must not become the runtime dependency. |
| YAML manifest -> validator | Human-editable YAML crosses into validation logic and must be schema-checked before later renderer use. |
| CLI args -> filesystem outputs | `--out-md`, `--out-json`, `--manifest`, and `--template` paths are untrusted command inputs. |
| PPTX XML -> report evidence | ZIP/XML content is parsed from an Office package and must be treated as data, not executable content. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-41-01 | Tampering | `standard-school.manifest.yaml` | mitigate | `template-report` validates exact layouts, slots, anchors, geometry tolerance, enum values, and text budget fields before accepting the manifest. |
| T-41-02 | Spoofing | theme selection | mitigate | Unknown themes fail non-zero with available controlled theme ids; only `standard-school` is accepted in Phase 41. |
| T-41-03 | Information Disclosure | source POTX path/provenance | accept | The source file is already inside the test workspace; reports should reference the normalized runtime template path and avoid embedding binary source contents. |
| T-41-04 | Denial of Service | large or malformed PPTX/ZIP | mitigate | Validator opens known skill-local template by default, catches ZIP/XML parse errors as validation failures, and writes bounded report evidence. |
| T-41-05 | Repudiation | manual template edits | mitigate | `template-editing.md` requires running `template-report` after drag/resize edits; JSON evidence records generated time, template path, manifest path, failures, and warnings. |
| T-41-06 | Elevation of Privilege | output path handling | mitigate | Script writes only caller-specified report paths, creates parent directories, does not execute Office macros, and does not shell out to untrusted paths. |
| T-41-SC | Tampering | npm/pip/cargo installs | mitigate | This plan does not include package-manager installs. If executor chooses to add one, stop and add package legitimacy audit plus blocking human verification before install. |
</threat_model>
