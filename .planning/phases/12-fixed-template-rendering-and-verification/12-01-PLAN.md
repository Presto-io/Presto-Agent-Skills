# Phase 12 Plan: Fixed-Template Rendering and Verification

## Goal

<objective>

Implement the rendering slice of the `end-of-term-teaching-materials` skill: fixed skill-local templates, a repeatable script entry point, structured-data-to-Markdown generation, Markdown-to-Typst rendering, merged PDF package compilation, deterministic table artifacts, and verification that supports later human format review.

</objective>

## Scope

In scope:

- Add skill-local fixed-template resources for the end-of-term package.
- Add sanitized fixture data and a sanitized workbook template fixture under the skill directory.
- Add a script entry point under `skills/end-of-term-teaching-materials/scripts/`.
- Support `example`, `validate`, `markdown`, `render`, `verify`, `manifest`, `info`, and `version` commands.
- Generate deterministic Markdown, Typst, manifest JSON, table JSON, CSV, teacher-facing workbook output, and a merged PDF package when `typst` is installed.
- Preserve blank score cells and declared task columns in all table artifacts.
- Respect `package` flags and skip `交接班记录封面` with a clear manifest warning when handover metadata is missing.
- Block final export verification whenever `## 复核标记` is not exactly `无`.
- Update runtime adapter notes for Codex, Claude Code, Gemini CLI, OpenCode, OpenClaw, and Hermes Agent.
- Update planning state and requirement checkboxes after verification.

Out of scope:

- OCR, handwriting recognition, photo extraction, or automatic extraction from arbitrary historical files.
- Arbitrary template discovery, marketplace template packs, or school-template selection.
- DOCX export or spreadsheet-native export beyond the fixed score-book workbook and deterministic companion table artifacts.
- Skipping the persistent Markdown checkpoint or rendering directly from raw input to PDF.
- Committing private real-course rosters, scores, source paths, credentials, or unsanitized user materials.

## Threat Model

<threat_model>

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Private student records or historical file paths leak into committed fixtures. | High | Use sanitized placeholder data; document that desktop materials are external validation references only. |
| Unresolved review markers are ignored and final exports are claimed ready. | High | Make `## 复核标记` export readiness a hard validation gate in `validate`, `render`, and `verify`. |
| Binary workbook comparison hides changed table data. | High | Emit deterministic JSON/CSV companion artifacts and verify them against Markdown rows. |
| Blank scores become zeros or review errors. | Medium | Preserve blanks in Markdown, CSV, JSON, and workbook output; include fixture assertions. |
| Missing Typst or workbook helper dependencies produce silent success. | Medium | Record explicit manifest statuses and fail verification when required local checks are unavailable without a declared skip. |
| Runtime-specific script behavior pollutes the canonical skill workflow. | Medium | Keep runtime differences in adapter notes; maintain one canonical `SKILL.md`. |

</threat_model>

## Execution Notes

- If a binary `.xlsx` fixture is generated during execution, commit only the sanitized reusable fixture and use deterministic companion artifacts for review.
- Update planning state only after the verification command passes.
