<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-authorized root → publisher | 根内可能含 hostile names、unknown files、symlink 与用户资料 |
| candidate → current/history | 未验证或混版候选不得跨入稳定交付集合 |
| audit plan → cleanup execution | 过期、篡改或过宽 approval 不得授权 mutation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-45-01 | Tampering | delivery root traversal/symlink | mitigate | component validation、no-follow inspection、identity checks、mutation 前 unknown gate |
| T-45-02 | Elevation of Privilege | cleanup authorization | mitigate | exact authorized roots、snapshot-bound approval、无默认 yes |
| T-45-03 | Denial of Service | stale lock/work | mitigate | 只清本 run 所有物；unrelated stale entry fail closed 并给 bounded diagnostic |
| T-45-04 | Repudiation | false atomicity claim | mitigate | 文档和测试限定 handled failure，明确硬崩溃非保证 |
| T-45-SC | Tampering | package installs | accept | 本计划不新增 npm/pip/cargo 依赖 |
</threat_model>
