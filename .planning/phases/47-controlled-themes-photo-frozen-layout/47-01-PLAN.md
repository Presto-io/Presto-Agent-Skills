## Trust Boundaries

| Boundary | Description |
|---|---|
| YAML photo -> assets root | 不可信路径只能成为受限普通本地文件。 |
| fonts manifest -> Typst | 字体文件、哈希和可见性共同定义可用字形。 |
| CLI projection -> user | 计划摘要不能成为照片路径、EXIF 或事实泄露通道。 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-47-01 | Information Disclosure | resolve_layout_photo | mitigate | POSIX 段校验、lstat/no-follow、root containment；统一 `PHOTO_ASSET_INVALID` 不回显绝对路径。 |
| T-47-02 | Tampering | fonts/manifest.json | mitigate | 相对路径 allowlist、精确哈希、字节数和 `typst fonts --ignore-system-fonts` 验证。 |
| T-47-03 | Information Disclosure | no-photo projection | mitigate | no-photo 不构造 PhotoAsset/slot，测试扫描 JSON 和 Typst 输入中媒体、EXIF、路径残留。 |
| T-47-04 | Spoofing | ThemeSpec | mitigate | ThemeSpec 无事实字段，固定主题/事实投影等价测试阻止主题伪造经历。 |
| T-47-SC | Tampering | local assets | mitigate | 未验证资源、symlink、非普通文件与系统字体均失败关闭。 |
</threat_model>
