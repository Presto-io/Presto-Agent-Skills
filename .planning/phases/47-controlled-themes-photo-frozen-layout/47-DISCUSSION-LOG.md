## 主题家族

| Option | Description | Selected |
|--------|-------------|----------|
| 内容导向分工 | 技术技能型、项目实践型、稳健通用型在内容强调上有明显分工 | |
| 样式导向差异 | 主题之间只做视觉和构图差异，不做内容差异 | ✓ |
| 自定义主题定位 | 用户直接定义三种主题的细粒度风格边界 | |

**User's choice:** 三种主题只是样式上的差异，而不是内容上的差异。  
**Notes:** 后续主题可以有明显不同构图，但不能通过切换事实顺序或强调对象制造差异。最终三分法锁定为 `保守稳妥 / 现代简洁 / 个性设计`。

---

## 照片版式

| Option | Description | Selected |
|--------|-------------|----------|
| 独立版式 | 有照片/无照片是同主题下两套独立构图 | |
| 固定占位可选元素 | 照片在固定槽位出现；无照片时按影响范围决定是否补位 | ✓ |
| 自定义处理 | 由用户为每个主题单独指定照片行为 | |

**User's choice:** 固定占位区域；没有照片时对应装饰也不显示，只有左侧或上部照片这类会影响主内容流的情况才需要正文补齐。  
**Notes:** 每个主题固定照片位置；照片默认优先不裁切，但保留受控裁切能力；照片装饰随主题变化但必须保持职业感。

## 页数策略

| Option | Description | Selected |
|--------|-------------|----------|
| 一页优先 | 先尽量挤进一页，再不行升到两页 | |
| 内容量判断 | 根据内容量决定 1 页或 2 页，不为页数硬凑 | ✓ |
| 两页优先 | 内容稍多就直接走 2 页 | |

**User's choice:** 根据内容量决定，不要为了 2 页而 2 页，也不要为了压缩到 1 页而 1 页。  
**Notes:** 自动模式后续细化为三段式：明显 1 页、明显 2 页、临界区。临界区需要给建议页数，同时生成双版本；自动模式尽量不失败，只有连两页也超边界时才失败。强制 1 页允许激进压缩，但要同时给出推荐 2 页版；强制 2 页但内容不足时要提示更适合 1 页。

## 冻结布局与分页单元

| Option | Description | Selected |
|--------|-------------|----------|
| 整个模块不可拆 | 项目、技能、证书等整个模块都不允许跨页 | |
| 条目优先完整 | 模块尽量不断，但必要时可拆；单个条目不可拆 | ✓ |
| 完全自适应 | 只要版面自然即可，条目也可拆开 | |

**User's choice:** 模块尽量不要拆，但实在不行可以拆开；单个条目不可拆。  
**Notes:** 核心目标是 HR 不用来回翻页才能看明白信息。用户进一步要求允许受限重排，但不能乱调，例如不能把个人信息挪到非首位；当分页困难时优先调模块顺序，再考虑压缩密度。

## 唯一事实源与通用容器

| Option | Description | Selected |
|--------|-------------|----------|
| 多份事实 MD | 1 页版、2 页版分别维护各自 Markdown | |
| 唯一事实源 | 同一份事实源派生所有页数、主题和照片版本 | ✓ |
| 混合模式 | 事实共享，但每个主题维护自己的中间稿 | |

**User's choice:** 一份 Markdown 作为唯一事实来源，后续所有版本都从这里重新生成。  
**Notes:** 用户提出使用一到两种通用容器承接不同模块，这样不怕将来增加个性化模块。最终记录为 `fact-block + list-entry`，由 agent 默认判断归属，但允许用户覆盖。

## 多主题预览原则

| Option | Description | Selected |
|--------|-------------|----------|
| 单一推荐 | agent 推荐 1 套主题并只生成那一套 | |
| 全量可见 | 默认让用户看到多主题，避免被少数模板限制 | ✓ |
| 只讨论后续 | 当前阶段不记录任何预览策略 | |

**User's choice:** 不要让用户被一两种模板样式限制死，应该有更多样式选择。  
**Notes:** 当前阶段只锁“支持多主题预览”的产品原则，以及预览可以是轻量预览的方向；具体是否 PDF、PNG 或 HTML，以及交付细节放到后续阶段处理。
