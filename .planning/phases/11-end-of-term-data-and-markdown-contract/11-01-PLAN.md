# Phase 11 Plan: End-of-Term Data and Markdown Contract

## Goal

<objective>

Create the first contract-only slice of the `end-of-term-teaching-materials` skill: a canonical skill file, a structured data contract, and a reusable Markdown intermediate template that lets semester-end teaching submission data be reviewed before any Phase 12 Excel, Typst, or PDF rendering.

</objective>

## Scope

In scope:

- Add `skills/end-of-term-teaching-materials/SKILL.md` as the canonical runtime-neutral skill source.
- Add `skills/end-of-term-teaching-materials/references/data-contract.md` for required and optional structured input fields.
- Add `skills/end-of-term-teaching-materials/templates/end-of-term-full.md` as the persistent teacher-editable Markdown intermediate.
- Define review-before-export behavior, including inline uncertain values such as `87?` and the mandatory `## 复核标记` section.
- Update repository skill listings so contributors can discover the new skill.
- Update planning state and requirement checkboxes after verification.

Out of scope:

- OCR, handwriting recognition, image extraction, spreadsheet ingestion, or roster ingestion implementation.
- Excel workbook export, Typst generation, PDF compilation, deterministic table artifact emission, or render scripts.
- DOCX or spreadsheet-native exports beyond the Phase 12 planning interface.
- Arbitrary template discovery, marketplace template packs, or direct source-to-PDF conversion.
- Derived Markdown fields such as `平时分`, `期末分`, or `学期成绩`; these stay renderer/formula responsibilities.

## Tasks

<tasks>

1. Define the canonical skill workflow.
   - Files: `skills/end-of-term-teaching-materials/SKILL.md`
   - Action: Start from the canonical skill template and write the trigger wording, objective, inputs, process, outputs, verification, success criteria, safety boundaries, and runtime adapter notes.
   - Must include:
     - Use cases for semester-end teaching submission documents and tables.
     - Expected explicit structured data inputs for course, class, teacher, semester, teaching-hour/task, assessment, and submission metadata.
     - The `structured data -> Markdown -> later fixed-template rendering` flow.
     - A hard rule that the final Markdown source has no `sources` frontmatter or hidden source list.
     - Review-loop behavior: do not claim final export readiness while `## 复核标记` contains anything other than `无`.
     - Adapter notes for Codex, Claude Code, Gemini CLI, OpenCode, OpenClaw, and Hermes Agent.
   - Verify: `rg` finds all required runtime names, `end-of-term-full.md`, `data-contract.md`, `## 复核标记`, and the no-final-export-until-review-cleared rule in `SKILL.md`.
   - Acceptance: ETM-01 is satisfied without introducing renderer implementation.

2. Write the structured data contract reference.
   - Files: `skills/end-of-term-teaching-materials/references/data-contract.md`
   - Action: Document required and optional fields for:
     - `template`, `date`, `school_year`, `semester`, `major_name`, `course_name`, `class_name`, and `teachers`.
     - Optional `handover_class_name` and `handover_teachers`.
     - `package` flags with Chinese artifact names: `成绩记分册`, `成绩汇总表`, `成绩分析表`, `教学日志封面`, `过程考核评价表封面`, and `交接班记录封面`.
     - Student roster entries, task/hour records, assessment columns, blank score cells, uncertain score values with `?`, analysis sections, and submission metadata.
   - Must include:
     - `class_name` and `handover_class_name` are arrays of short class names; renderers add `班` where target templates require it.
     - `teachers` is an ordered array; do not add `primary_teacher`.
     - Missing `package` keys default to `true`, except handover output requires handover metadata.
     - `成绩记分册: true` is one bundle for the redesigned cover plus score-book body.
     - Blank score cells remain blank and are not `0`, `-1`, or automatic review items.
     - Values like `87?` are uncertain and must be mirrored in `## 复核标记`.
   - Verify: Content checks confirm every ETM-02 field category and every Phase 11 package key appears in the reference.
   - Acceptance: ETM-02 is satisfied through a skill-local, reviewable contract.

3. Create the reusable Markdown intermediate template.
   - Files: `skills/end-of-term-teaching-materials/templates/end-of-term-full.md`
   - Action: Write a complete example Markdown package with YAML frontmatter and body sections:
     - `## 我带的学生`
     - `## 过程考核任务`
     - `## 成绩数据`
     - `## 分析`
     - `### 试卷分析`
     - `### 存在问题`
     - `### 今后改进措施`
     - `### 异常情况分析`
     - `## 复核标记`
   - Must include:
     - Fully padded and aligned Markdown tables for score data and review markers.
     - Task columns that map to the ordered `## 过程考核任务` list.
     - At least one blank score cell to prove blanks are preserved.
     - At least one uncertain value such as `87?` plus a matching review item.
     - AI draft markers for analysis text where source text is absent.
     - No `sources` entry and no derived score fields.
   - Verify: Markdown inspection confirms the required headings, table columns, review marker mirror, and absence of `平时分`, `期末分`, `学期成绩`, and `sources`.
   - Acceptance: ETM-03 and ETM-04 are satisfied by a durable, teacher-reviewable source of truth.

4. Connect the review contract across skill and template.
   - Files:
     - `skills/end-of-term-teaching-materials/SKILL.md`
     - `skills/end-of-term-teaching-materials/references/data-contract.md`
     - `skills/end-of-term-teaching-materials/templates/end-of-term-full.md`
   - Action: Make the review loop executable for an agent by documenting the one-by-one confirmation question shape and the required edit-after-answer behavior.
   - Must include:
     - Review questions name the student, task or field, and current value.
     - After confirmation or correction, the agent edits the Markdown value and removes the corresponding review item.
     - When review is cleared, `## 复核标记` body is exactly `无`.
     - At the end of a run, the agent reports remaining review items, or says review is cleared.
   - Verify: Cross-file checks confirm the same semantics appear in the skill process and the data contract, and the template demonstrates the unresolved state.
   - Acceptance: ETM-04 is covered as behavior, not just syntax.

5. Update contributor-facing listings.
   - Files:
     - `README.md`
     - `skills/README.md`
   - Action: Add `end-of-term-teaching-materials` to the existing skill list and update the repository tree only as needed.
   - Verify: The README entries remain concise and do not claim Phase 12 rendering exists yet.
   - Acceptance: Contributors can discover the new skill without confusing Phase 11 contract work with Phase 12 outputs.

6. Verify and close planning state.
   - Files:
     - `.planning/REQUIREMENTS.md`
     - `.planning/ROADMAP.md`

   - Verify: `git diff --check` passes; requirement coverage shows ETM-01 through ETM-04 complete and ETM-05 through ETM-08 still planned.
   - Acceptance: The milestone state accurately reflects a completed contract slice and does not imply renderer completion.

</tasks>

## Success Criteria

<success_criteria>

- `skills/end-of-term-teaching-materials/SKILL.md` defines when to use the skill, expected source data, safety boundaries, verification path, and adapter notes for all required runtimes.
- The skill directory includes a structured data contract covering course, class, teacher, semester, teaching-hour records, assessment records, and submission metadata.
- `skills/end-of-term-teaching-materials/templates/end-of-term-full.md` provides one reusable Markdown intermediate with separate reviewable document/table sections.
- Missing, ambiguous, or lossy teaching data is visible through inline uncertainty markers and `## 复核标记`; unresolved review items block final export readiness.
- Phase 11 does not implement Phase 12 render scripts or claim Excel/PDF export is complete.

</success_criteria>

## Threat Model

<threat_model>

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Private student scores or roster data leak into reusable examples. | High | Use placeholder data in templates and avoid credentials, private paths, or real source lists in reusable artifacts. |
| Agent silently guesses uncertain handwriting or missing score data. | High | Require inline `?` uncertainty markers and matching `## 复核标记` rows; block final export readiness until review is cleared. |
| Blank cells are converted into zeros or review errors. | Medium | Contract states blank score cells remain blank and are not automatic review items. |
| Phase 11 accidentally ships renderer behavior before the contract is reviewed. | Medium | Plan excludes scripts, Excel, Typst, PDF, OCR, and deterministic table artifact implementation. |
| Runtime-specific tool behavior pollutes the canonical skill body. | Medium | Keep runtime differences in `Runtime Adapter Notes` and keep the main workflow runtime-neutral. |

</threat_model>

## Execution Notes

- Keep files concise and copyable; put lengthy field detail in `references/data-contract.md`.
- Do not create runtime-specific wrapper files.
- Do not add scripts in Phase 11 unless the roadmap is explicitly changed.
- Prefer placeholder student/class data in the reusable template. If real teacher-provided data is needed for later black-box verification, put it in Phase 12 fixtures only after confirming the privacy boundary.
