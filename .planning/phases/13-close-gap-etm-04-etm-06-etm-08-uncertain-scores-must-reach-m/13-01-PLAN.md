# Phase 13 Plan: Uncertain Scores Must Reach Markdown Review

## Goal

<objective>

Close the v1.7 audit blocker by allowing structured uncertain score values such as `87?` to generate reviewable Markdown with matching `## 复核标记` rows, while preserving strict final export readiness and adding explicit abnormal preview artifacts with red warning styling when the user chooses to inspect unresolved data.

</objective>

## Scope

In scope:

- Keep uncertain score data represented as inline strings such as `87?`.
- Allow `markdown` generation from structured JSON containing unresolved uncertain scores.
- Auto-create review markers when uncertain score values appear without a matching marker.
- Keep blank score cells blank and out of automatic review markers.
- Keep final export readiness strict for normal `validate`, `render`, and verification flows.
- Add an explicit abnormal preview path for unresolved review artifacts.
- Highlight unresolved uncertain score cells in abnormal workbook/PDF artifacts.
- Highlight generated `学期成绩` values below 60 in score-book workbook/PDF outputs where that field is shown.
- Extend black-box verification to cover uncertain source JSON, final-export blocking, abnormal preview status, red-highlight evidence, and audit metadata.
- Normalize the missing GSD verification metadata identified by the v1.7 audit.

Out of scope:

- OCR, handwriting recognition, or image-to-structured-data extraction.
- Replacing `87?` with nested score objects.
- Redesigning the Phase 11 Markdown contract or splitting the single Markdown source of truth.
- Redesigning fixed templates or adding arbitrary template discovery.
- Treating abnormal preview artifacts as final submission-ready outputs.
- Committing private student records, raw historical materials, private file paths, credentials, or unsanitized source data.

## Threat Model

<threat_model>

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Uncertain source data is rejected before the teacher can review it. | High | Permit `?` scores only in review-generation mode and verify Markdown contains the inline value plus review marker. |
| Abnormal preview output is mistaken for a final-ready submission package. | High | Require explicit abnormal flag, set manifest `final_ready=false`, and document user confirmation before generation. |
| Final export readiness becomes too permissive. | High | Keep normal render/export validation strict and add a negative unresolved-review test. |
| Auto-created markers do not tell the teacher what to confirm. | Medium | Include student, task/field, current value, and reason in every generated marker. |
| Blank cells become errors or zeros. | Medium | Preserve blanks and assert they are not automatic review markers. |
| Red warning styling is visually present but untestable. | Medium | Add deterministic manifest/table highlight metadata in addition to workbook/PDF styling. |
| Private score data leaks into new fixtures. | High | Use sanitized placeholder data and run manual fixture review. |
| Audit cleanup expands beyond the blocker. | Medium | Limit metadata normalization to Phase 11 verification, Phase 12 summary frontmatter, and Phase 13 verification. |

</threat_model>
