## Trust Boundaries

| Boundary | Untrusted Input / Abuse Path | Structural Control |
|----------|------------------------------|--------------------|
| target runtime → installation record | 调用者可在同一 shell 改标签伪造六环境，或用文档行冒充实际执行 | exactly-six fixed IDs、真实 executable/version/environment、whole-folder manifest、raw command/artifact hashes 与 cross-record uniqueness checks |
| installed folder → public CLI | 缺 support、路径逃逸、symlink/unknown、私有 wrapper 可偏离 canonical workflow | whole-folder manifest、relative no-follow support checks、唯一 canonical SKILL、explicit `graduate-resume.sh` fallback |
| automation → human UAT record | 自动化可能填写 tester/time/observation/PASS，自批视觉结论 | prepare 仅生成全 PENDING；lint read-only；人工字段非空与 signoff 必须来自外部记录，mutation guards 禁止自动值 |
| acceptance artifacts → stale UAT | PDF/PNG/font/fixture/Typst/runtime/environment 变化后旧签字可能被沿用 | exact SHA-256 cross-binding；任一变化即 stale failed，必须重新人工验收 |
| active-run locator → runtime/UAT continuation | 旧/错误 run ID、silent new run 或跨 run record 可把签署绑定到其他 artifacts | 显式 `--active-run-id`/`--resume`、locator/run-local acceptance/held identity 复验、G11/G13 仅消费同 run exact hashes |
| G11/G13 → final release arbiter | runtime 缺失或人工未签可能被宽泛 human_needed/PASS 掩盖 | 唯一三态矩阵：仅 G13 合法 PENDING 可 human_needed；任何其他缺项 failed；全 14 gate 通过才 passed |
