<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Untrusted YAML/Markdown -> logical model | Implicit YAML types, malformed tables, and directive-like code text must not corrupt semantics, escape JSON safety, or alter slide structure. |
| Manifest bytes -> validator state | A missing, malformed, or structurally invalid manifest is untrusted configuration and must not leave partially initialized state or disclose a traceback. |
| Caller-controlled `--out-dir` -> fixed example writer | An attacker may replace either the output root or `media` parent with a symlink between checks and publication; a runtime may also lack a descriptor primitive required to close that race. |
| Logical model/diagnostics -> JSON and terminal | Serialization and failure reporting must remain deterministic, bounded, truthful, and free of source/exception disclosure. |
| Regression harness -> completion claim | Every reproduced gap must be exercised by the authoritative command before the phase can be declared closed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-42-17 | Tampering | YAML metadata | mitigate | Tasks 1 and 3 accept only composed string scalars, reject implicit typed values with `YAML_VALUE_TYPE`, and compare validation with/without JSON. |
| T-42-18 | Tampering | slide container state | mitigate | Tasks 1 and 3 track matching fence markers before directives/headings and preserve directive-like lines as opaque code payload. |
| T-42-19 | Tampering | table/timeline model | mitigate | Tasks 1 and 3 require a valid separator and equal row widths before accepted logical table semantics reach Phase 43. |
| T-42-20 | Information Disclosure | manifest/internal failures | mitigate | Tasks 2 and 3 initialize safe state, map three manifest failure classes to bounded diagnostics, catch serialization failures, and prohibit traceback/source dumps. |
| T-42-21 | Elevation of Privilege | example root/parent symlink exchange | mitigate | Tasks 2 and 3 preflight all required descriptor/no-follow/dir-fd capabilities, fail closed before writing when any are absent, hold root/parent directory descriptors, publish relative to those handles, verify pathname/inode identity, and assert both root-exchange and media-exchange outside targets remain unchanged. |
| T-42-22 | Repudiation | gap closure claim | mitigate | Task 3 makes all five reproductions mandatory under `fixture-example` and preserves existing canonical, ownership, resource, and Phase 41 assertions. |
| T-42-23 | Denial of Service | malformed input/failure output | mitigate | Tasks 1-3 retain existing input/diagnostic limits and cap new manifest, type, table, and race diagnostics below 4,000 terminal characters per case. |
| T-42-SC | Tampering | package installation | mitigate | This plan adds no package-manager dependency; any proposed install stops execution until legitimacy research and a blocking human check are added. |
</threat_model>
