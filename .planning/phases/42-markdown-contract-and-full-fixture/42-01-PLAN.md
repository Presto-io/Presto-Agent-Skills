<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Untrusted Markdown/YAML -> parser | Teacher/agent-authored text may contain malformed containers, hostile YAML, raw HTML, style controls, oversized blocks, or misleading fences. |
| Markdown media path -> filesystem | Relative and absolute media paths may traverse directories, point to non-files, disclose machine paths, or trigger unintended fallback/search behavior. |
| Manifest -> contract validator | Human-editable YAML defines the trusted controlled theme/layout/slot boundary and must not be duplicated or silently weakened. |
| Parser diagnostics/JSON -> caller | Errors may expose absolute paths, stack traces, source contents, or nondeterministic machine details. |
| CLI output path -> filesystem | `--out-json` is caller-controlled and must not overwrite inputs, directories, or unrelated paths implicitly. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-42-01 | Tampering | Markdown/YAML state machine | mitigate | Tasks 2-3 apply strict allowlists, safe YAML loading, explicit container/fence states, aggregate structural validation, and invalid non-zero exits; malformed-input cases verify behavior. |
| T-42-02 | Elevation of Privilege | raw HTML/style/coordinate/font/color syntax | mitigate | Tasks 1-3 forbid all non-contract directives and attributes outside fenced code, never render/execute HTML, and verify fence-aware rejection. |
| T-42-03 | Information Disclosure | relative/absolute media resolution | mitigate | Task 2 resolves relative paths only from the input parent, preserves authored paths, performs file checks without network/CWD/sibling fallback, and avoids reading media bytes into diagnostics. |
| T-42-04 | Tampering | path traversal and output collision | mitigate | Tasks 2-3 normalize paths, keep media read-only, reject non-file media, require explicit `--out-json`, refuse input/output identity and directory collisions, and never delete parent directories. |
| T-42-05 | Information Disclosure | diagnostics and internal exceptions | mitigate | Tasks 2-3 emit bounded location/code/message/fix fields, suppress default stack traces and source dumps, and avoid machine-specific paths in deterministic JSON where not required for resolution. |
| T-42-06 | Denial of Service | oversized or malformed Markdown/YAML | mitigate | Tasks 2-3 enforce and exercise fixed limits of 2 MiB input, 64 KiB frontmatter, 32 KiB lines, 2,048 blocks, depth 8, 256 YAML nodes, zero YAML aliases, and 200 collected diagnostics; every breach returns a stable bounded non-zero diagnostic without stack/source dumps. |
| T-42-07 | Repudiation | validation outcome | mitigate | Tasks 2-3 keep `errors` in JSON, sort diagnostics deterministically, and return 0 only when `errors == []`; placeholder metadata never converts invalid input into success. |
| T-42-08 | Spoofing | controlled theme/layout ids | mitigate | Task 2 derives themes and authorability from the Phase 41 manifest and rejects explicit fixed `closing`; no duplicate layout registry is trusted. |
| T-42-SC | Tampering | npm/pip/cargo installs | mitigate | This plan adds no package-manager install. Any proposed install stops execution until RESEARCH contains a package legitimacy audit and a blocking human legitimacy check is added. |
