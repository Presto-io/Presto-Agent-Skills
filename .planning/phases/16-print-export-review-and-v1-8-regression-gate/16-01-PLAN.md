# Phase 16 Plan: Print Export Review and v1.8 Regression Gate

## Goal

<objective>

Deliver the Phase 16 print/export review surface and final v1.8 regression gate for `school-presentation`: an explicit review/print mode inside the existing offline single-file HTML deck where reveal, mask, emphasis, sorting, section dividers, inserted physical pages, gallery pagination, and fixed `16:9` / `4:3` canvases can be inspected outside live playback, while preserving Markdown as the source of truth and keeping presenter annotations playback-local by default.

</objective>

## Scope

In scope:

- Formalize review/print mode in the generated offline HTML deck, using browser print/export to PDF for human UAT.
- Print/review all reveal content opened, all masks visible, emphasis underlines visible, and sorting exercises in correct final order.
- Keep section title pages visible by default during live playback and workspace review.
- Add generated-deck controls for including or excluding section title pages from print/PDF review.
- Strengthen print CSS and DOM hooks for inserted physical pages, section dividers, gallery card-level pagination, and fixed canvas stability.
- Exclude presenter-local annotations from print/export review, manifest output, deterministic review artifacts, and Markdown source.
- Extend `verify` with print/review checks and a full v1.8 regression gate covering Phase 14, Phase 15, offline single-file, and fixed ratio behavior.
- Update canonical docs, fixture, and planning closeout for SP-26 and SP-27.

Out of scope:

- Separate PDF renderer, PPTX export, Keynote export, editable PowerPoint output, office-suite automation, or hosted sharing.
- Playwright, browser screenshot checks, or automated PDF generation as part of the default verification gate.
- Annotation flattening, saved annotation export, annotation persistence, remote control, audience synchronization, or multi-device collaboration.
- Drag-and-drop sorting, arbitrary animation builder, raw HTML authoring, external icon libraries, CDN dependencies, or runtime-specific wrapper syntax.

## Threat Model

<threat_model>

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Review/print mode only works incidentally through preview CSS and lacks a clear user entry point. | High | Add explicit generated-deck controls and verify rendered review/print tokens. |
| Printed output hides reveal or mask content that a reviewer needs to inspect. | High | Add print/review CSS forcing reveal and mask visibility, with token and manual PDF checks. |
| Sorting prints candidate order instead of final order. | High | Add final-order print semantics and fixture checks for `sort-final` / `is-sorted`. |
| Presenter annotation state leaks into print/PDF, manifest, or deterministic artifacts. | High | Exclude annotation layers in print and extend manifest leakage checks. |
| Section divider inclusion/exclusion reorders pages or hides required inserted pages. | High | Keep manifest physical-page order as the truth source and verify section/insertion tokens. |
| Gallery cards are clipped by browser print pagination. | Medium | Add card-level `break-inside` / `page-break-inside` rules and manual PDF UAT focus. |
| Fixed `16:9` / `4:3` canvas behavior regresses through print-specific CSS. | High | Preserve design-dimension CSS, verify ratio tokens, and add a 4:3 render sample if needed. |
| Verification becomes too heavy or environment-specific. | Medium | Keep default gate deterministic and script-based; leave browser PDF as manual UAT. |
| Phase 16 expands into PPTX, Keynote, hosted sharing, or annotation export. | Medium | Preserve explicit out-of-scope list in plan, docs, and final summary. |

</threat_model>
