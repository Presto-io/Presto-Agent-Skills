# Plan 18-01: Decompose Large Script Entry Points

<objective>
Split the largest current script entry points into skill-local modules with clear ownership boundaries while preserving public command names, common flags, default output paths, generated artifact names, manifest keys, Markdown intermediate contracts, user-facing invocation patterns, and runtime-neutral script interfaces.
</objective>

<context>
- Read `AGENTS.md` before editing.
- Read `.planning/phases/18-script-module-decomposition/18-CONTEXT.md`, `18-RESEARCH.md`, and `18-PATTERNS.md`.
- Read `.planning/ROADMAP.md`, `.planning/REQUIREMENTS.md`, and `.planning/STATE.md`.
- Read Phase 17 summary and verification to confirm script internals were intentionally left for Phase 18.
- This plan covers SD-05 through SD-08 only.
- Do not perform Phase 19 contributor documentation review, milestone audit, milestone closeout, or broad feature expansion.
</context>

<threat_model>
This phase does not add network access, credential handling, database schema changes, or new external services. The main risks are local file write regressions, path traversal regressions in asset handling, and accidental execution/runtime dependency changes. Mitigate by preserving existing path guards, keeping new modules skill-local, refusing parent-directory escapes where existing code refuses them, and verifying all behavior through public shell commands.
</threat_model>
