# Phase 15 Plan: Classroom Interaction and Structured Layout Extensions

## Goal

<objective>

Deliver the Phase 10 deferred classroom-interaction and structured-layout authoring surface for `school-presentation`: Markdown block directives for peek hints, sorting exercises, restrained body-step animation, timeline/cards/gallery/smartart layouts, semantic icons, and section-divider controls, while preserving the existing Markdown source of truth, offline single-file HTML output, fixed canvas model, reveal/mask/emphasis behavior, overview/workspace/playback modes, and Phase 14 playback-local presenter markup boundaries.

</objective>

## Scope

In scope:

- Add or connect Markdown block directives: `::: peek :::`, `::: timeline :::`, `::: cards :::`, `::: gallery :::`, and `::: smartart type=... :::`.
- Preserve the existing sorting model: Markdown order is the initial candidate order, item `order` values reveal rank numbers, and `final_order` reveals the final sorted state.
- Keep `animate: step` restrained to ordinary paragraphs, list items, and table rows; do not automatically animate structured-layout items.
- Render timeline, cards, gallery/compare, and smartart/process-like layouts with the existing blue-green school identity CSS.
- Enable semantic icons by default for ordinary content slides and structured-layout items, with `icon=none` as the opt-out.
- Document section-divider/section-cover controls and verify the existing hide/show behavior.
- Extend the fixture and verification manifest so the user can run the script gate and then manually inspect generated artifacts.

Out of scope:

- Phase 16 print/export review mode and full v1.8 regression packaging.
- Drag-and-drop sorting, audience participation, remote control, hosted collaboration, annotation persistence, or multi-device synchronization.
- PPTX, Keynote, or editable PowerPoint export.
- Raw HTML authoring or runtime-specific wrapper syntax.
- External icon libraries, CDNs, browser screenshot default gates, or network dependencies.

## Threat Model

<threat_model>

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Dormant structure helpers remain disconnected while docs claim support. | High | Add parser/render routing and verify rendered fixture tokens for each directive family. |
| Peek hover/click steals deterministic reveal or mask interactions. | High | Keep reveal advancement click-based, use explicit peek trigger hit-testing, and preserve mask/reveal handlers in verification. |
| Presenter markup gestures conflict with peek or click zones. | High | Preserve Phase 14 `isMarkupInputTool()` priority and token-check event arbitration. |
| `animate: step` makes structured pages noisy or unreadable. | Medium | Limit auto animation to ordinary paragraphs, list items, and table rows; keep structures static unless explicitly wrapped. |
| Semantic icons make formal school pages feel decorative or inconsistent. | Medium | Use restrained skill-local CSS icons, disable automatic icons on cover/closing/section pages, and support `icon=none`. |
| Structured layouts overflow fixed slide canvas or break thumbnails. | Medium | Keep stable dimensions, reuse existing fixed-canvas CSS, and verify thumbnail ratio plus manual visual review artifacts. |
| Runtime-only state leaks into manifest or source artifacts. | High | Add manifest leakage checks for annotation and peek runtime state. |
| Phase 15 accidentally expands into Phase 16 print/export scope. | Medium | Document Phase 16 as deferred and only keep static fallback readable in this phase. |

</threat_model>
