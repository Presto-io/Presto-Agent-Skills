# Phase 14 Plan: Presenter Markup Tools

## Goal

<objective>

Add playback-local presenter markup tools to the existing `school-presentation` offline HTML deck: pointer, pen, highlighter, eraser, and clear/reset controls that persist per physical page for the active session while preserving Markdown source, manifest hierarchy, preview workspace, deterministic review output, navigation, ordered reveals, emphasis animation, and answer masks.

</objective>

## Scope

In scope:

- Add a compact iPad-style markup palette inside playback only.
- Support pointer, pen, highlighter, eraser, clear current page, and palette placement controls.
- Store annotations in session memory keyed by physical `page_id`; preserve them across page navigation until explicit clear/reset.
- Render annotations in slide design coordinates so strokes remain aligned under `16:9`/`4:3`, browser resizing, and stage scaling.
- Ensure drawing/erasing gestures win over playback click zones while keyboard navigation remains available.
- Extend deterministic verification to prove markup controls exist and previous Phase 9/10 behavior still passes.
- Update `school-presentation` docs/template examples to describe playback-local markup without creating authored annotation syntax.

Out of scope:

- Saving annotations back into Markdown, source frontmatter, manifest hierarchy, or preview workspace content.
- Hosted collaboration, remote control, audience sync, multi-device pointer sharing, or persistent annotation storage.
- Print/export annotation flattening or exclusion policy beyond keeping Phase 14 annotations out of deterministic review output.
- New Phase 15 authoring syntax for peek, sorting, body steps, timelines, cards, gallery/compare, process layouts, icons, or section-cover controls.
- PPTX, Keynote, or editable PowerPoint export.

## Threat Model

<threat_model>

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Presenter annotations accidentally become authored Markdown or manifest data. | High | Keep state in runtime memory, render only inside playback shell, and add manifest leakage checks. |
| Drawing clicks advance slides or reveal answers unexpectedly. | High | Suppress playback zones and stop propagation while pen/highlighter/eraser gestures are active; keep keyboard navigation as the escape path. |
| Annotation strokes drift under stage scaling, resize, or `4:3` output. | High | Store coordinates in slide design space and render the SVG layer with the same scale mechanics as `.slide`. |
| Page navigation loses presenter marks. | Medium | Key annotation state by `page_id`; redraw on `selectPage()` and preserve state until explicit clear. |
| Markup UI obscures or destabilizes existing playback controls. | Medium | Use compact placement families and verify toolbar/zone tokens still exist. |
| Touch and pointer events break mask reveal or peek behavior. | Medium | Route drawing gestures only when drawing tools are active; leave non-drawing playback events on existing handlers. |

</threat_model>
