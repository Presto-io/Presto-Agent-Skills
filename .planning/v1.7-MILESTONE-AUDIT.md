# Milestone v1.7 Audit: End-of-Term Teaching Materials Skill

## Verdict

**Status: gaps_found**

The v1.7 happy path is mostly wired: the skill can validate a clean source fixture, generate Markdown, render Typst/PDF, emit deterministic JSON/CSV table artifacts, create `scorebook.xlsx`, and expose runtime adapter notes for Codex, Claude Code, Gemini CLI, OpenCode, OpenClaw, and Hermes Agent.

However, the milestone is not ready to archive because a Phase 11 contract behavior is not integrated into Phase 12 generation: `87?` uncertain score values are supposed to reach Markdown with matching `## 复核标记` rows, but the structured JSON -> Markdown command rejects them before review.

## Broken Flow

**Structured data with uncertain score -> Markdown review -> corrected export**

Phase 11 defines `87?` as a reviewable uncertain value and requires a matching `## 复核标记` entry. This is the right contract for teacher review. Phase 12 then calls validation before Markdown generation, but the validator rejects any score value containing `?` even when `export_ready=False`.

Result: an agent cannot generate the reviewable Markdown intermediate from structured input when a score is uncertain. That breaks the milestone's core review-before-export behavior for ambiguous data.
