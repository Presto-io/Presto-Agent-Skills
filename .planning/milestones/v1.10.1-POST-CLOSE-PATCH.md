# v1.10.1 Post-Close Patch

## Purpose

This patch records a post-close visual and black-box validation issue found after the v1.10 milestone was archived. The fix stays inside the shipped v1.10 scope: `jiaoan-shicao` Markdown-to-Typst generation must match the official Presto black-box template for the same Markdown input.

This patch does not reopen the v1.10 milestone and does not introduce a new feature milestone.

## Included Debug Session

| Debug | Scope | Resolution | Commit |
| --- | --- | --- | --- |
| `test/validation/blackbox-jiaoan-shicao/DEBUG.md` | Align `jiaoan-shicao` learning-resource and teaching-activity table wrapping with the official black-box template. | Resolved | `144df5f` |

## Root Cause

The post-close manual review found visible line-wrapping differences in:

- the `学习资源` table, where the three resource columns used old approximate widths; and
- the `教学活动设计` table, where activity columns did not use the official content-driven width distribution.

The skill-local `render_v110_typst.awk` renderer also emitted an empty document author when `teacher_name` was blank, while the official template falls back to `Presto`.

## Fix

`skills/jiaoan-shicao/scripts/render_v110_typst.awk` now mirrors the official `presto-official-templates/jiaoan-shicao/main.go` behavior for the post-close validation case:

- empty `teacher_name` falls back to `author: "Presto"`;
- `学习资源` column widths use the official content-driven 0.02cm search strategy; and
- `教学活动设计` column widths use the official pressure and weight distribution, shared per chapter.
