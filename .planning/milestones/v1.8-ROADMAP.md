## Milestone Summary

**Key Decisions:**

- Presenter markup remains playback-local session state only; no annotation data is written back to Markdown, manifests, preview, overview, thumbnails, or deterministic review artifacts.
- Structured classroom syntax stays teacher-facing: `peek`, `timeline`, `cards`, `gallery`, and `smartart`, with raw HTML and separate `process` directives avoided.
- Print/export review extends the same offline HTML deck instead of introducing a separate PDF renderer or PPTX/Keynote path.
- One-click final PDF export is generated inside the offline deck because browser print could not reliably satisfy the accepted outline/bookmark and agenda-link workflow.

**Issues Resolved:**

- Presenter-toolbar keyboard focus and stroke-capture conflicts.
- Presenter-toolbar animation, collapse, placement, and edge-switching defects.
- Unknown structured directives no longer silently drop author content.
- Print CSS no longer produces letter/portrait whitespace; ratio-specific print geometry is generated.
- Section divider pages are real generated physical pages.
- Final PDF export now produces reader outline/bookmarks and agenda links.

**Issues Deferred:**

- PPTX, Keynote, and fully editable PowerPoint export.
- Hosted sharing, password-gated distribution, multi-device synchronization, remote control, and audience participation.
- Persistent saved annotations and annotation flattening.
- Selectable/copyable live text in the one-click PDF export; the accepted contract uses rasterized slide canvases for reliability.

**Technical Debt:**

- PDF outline/link validation is recorded through Chrome/manual UAT evidence, while the default deterministic script gate stays HTML/manifest based.
- The verification fixture intentionally includes missing-image and missing-video fallback cases, so media fallback warnings are expected and non-blocking.
